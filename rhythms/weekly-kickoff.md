# Rhythm: Weekly Kickoff (Monday macro)

Run this at the start of the user's work week. Goal: ground the week in the quarter, name 3–5 weekly goals, and pre-shape the week's calendar.

## Pre-read (run first, silently)

- Load `user-profile.md`, `relationship-map.md`, `goals.md`, `todos.md`, `time-log.md` (last week), `calibration-log.md`
- Pull this week's calendar via MCP
- Pull inbox + Slack unread / unactioned from over the weekend

## Step 1 — Anchor to the quarter

Show the user their quarterly goals from `goals.md`. Ask:

> "Before we set this week's goals — has anything about the quarter changed since we last talked? New priorities, things that shifted, goals that need updating?"

If yes → update `goals.md` before proceeding. If no → move on.

Then, for each quarterly goal, show a rough status:

- On track / at risk / behind / ahead
- What was done toward it last week (from `time-log.md` + last week's closed items in `todos.md`)

## Step 2 — Set this week's goals (3–5)

Ask:

> "What are your 3–5 goals for this week? Each one should ladder up to a quarterly goal."

For each proposed weekly goal:

- Confirm which quarterly goal it ladders to
- Confirm it's a concrete outcome, not a topic
- If it doesn't ladder → ask whether to cut it or update the quarterly goals

Write them into `goals.md` under `## Week of YYYY-MM-DD`.

## Step 3 — The week's keystone

Ask:

> "Of these goals, which one, if nailed, makes the rest easier or less necessary?"

This is the week's keystone. Note it in `goals.md`. The daily kickoff will preferentially pull MITs from this.

## Step 4 — Calendar shape

Surface:

- Total meeting hours this week
- Days with zero focus block available
- Overload days (>5 meeting hours)
- Non-negotiables holding or violated
- Missing prep on any meetings

Propose changes — declines, reschedules, focus blocks — tied to the week's goals. Confirm each with the user before executing via calendar MCP.

## Step 5 — People sweep

From `relationship-map.md`, surface:

- Scheduled 1:1s this week (flag any missing context packs to prep)
- Direct reports not 1:1'd in ≥2 weeks (propose slotting one in)
- Key stakeholders gone quiet (propose a touchpoint)

## Step 6 — Backlog triage

Quick pass on `todos.md`:

- Anything in `# Waiting On` >7 days → nudge
- Anything in `# Backlog` quadrant Q1 that didn't become an MIT → discuss
- Anything in Q4 older than 14 days → propose delete

## Step 7 — Seed tomorrow's daily kickoff

Pick candidate MITs for Monday (3–5, pulled from the keystone + other weekly goals). Write them into `todos.md` under `# Today's MITs`. The Monday daily kickoff will confirm or adjust.

## Close-out

Summarize for the user:

- The 3–5 weekly goals
- The keystone
- Major calendar changes made
- People touchpoints slated
- Candidate MITs for Monday

Ask: "Anything I missed or got wrong?"

Write all updates back to the relevant context files before ending.
