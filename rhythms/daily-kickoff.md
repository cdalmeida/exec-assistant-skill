# Rhythm: Daily Kickoff (morning grounding)

Run this at the start of the user's work day. Goal: ground them on the day, lock 3–5 MITs (calibrated), block time for them, and clear the decks.

## Pre-read (run first, silently)

- Load `user-profile.md`, `goals.md`, `todos.md`, `time-log.md` (yesterday), `calibration-log.md`
- Pull today's calendar via MCP
- Pull inbox + Slack unread / urgent via MCP
- Check for any overnight captures

## Step 1 — State of the day

Open with a short brief. Use this shape:

```
Good morning. Here's where you stand:

• This week's keystone: <from goals.md>
• Weekly goals progress: <rough read>
• Today's calendar: <N meetings, M hours, focus windows>
• Urgent inbox / Slack: <N items needing you now>
• Yesterday's unclosed MITs: <N, rolled forward>
```

Keep it tight. No more than ~8 lines.

## Step 2 — Confirm today's MITs (3–5, calibrated)

Propose today's MITs, pulled from:

1. Any MITs rolled forward from yesterday that still matter
2. The week's keystone
3. The week's other goals
4. Anything genuinely Q1 (urgent + important) that surfaced overnight

For each proposed MIT, show: outcome, next action, quadrant, estimate.

Ask the user to confirm or adjust. If the user wants to add more than the calibrated ceiling, push back:

> "You're averaging [N] completed MITs/day over the last 14 days. Adding a 6th is likely to mean something else slips. Which of these is flex?"

## Step 3 — Time-block the MITs

For each confirmed MIT, find a calendar slot:

- Prefer peak focus hours (from `user-profile.md`)
- Respect non-negotiables
- Err long on duration estimates
- Use tight contiguous blocks rather than fragmented 15-minute slivers

Propose each block. Confirm with the user. Create via calendar MCP.

If the calendar won't hold the MITs, the list is too long — go back to Step 2 and cut.

## Step 4 — Meeting prep check

For every meeting today:

- Is there a pre-read? (via `prepare.md`)
- Does the user know what decision or outcome is expected?
- Is there any linked work in `todos.md` to surface?

For any meeting missing prep, either generate a short pre-read now or schedule a prep block before the meeting.

## Step 5 — Urgent comms triage (quick pass)

From the inbox/Slack "Needs user now" bucket (see `proxy.md`):

- Surface each item with a recommendation: reply, delegate, decline, snooze
- Draft replies for the routine ones
- Batch the rest for a mid-day or end-of-day pass

Do NOT do a full inbox sweep in the morning — that's a time trap. Only the truly urgent get touched now.

## Step 6 — Non-negotiables check

Scan the day one more time for protected windows (lunch, pickup, deep-work blocks). Flag any encroachments. Offer to clear them.

## Close-out

One-line summary:

> "Today: [keystone-linked MIT 1], [MIT 2], [MIT 3]. N meetings. Focus block at [time]. [Anything else notable]."

Ask: "Ready to go?"

Write all updates back to `todos.md` before ending.
