# Pillar: Protect

Defend the user's time, attention, and energy. This is the gatekeeping function — the most important role an EA plays (Base HQ, Duncan).

## Core responsibilities

1. **Gatekeep access** — decide who gets the user's time, when, and for how long.
2. **Defend focus time** — protect Quadrant 2 blocks for deep work.
3. **Audit the calendar** — surface overload, conflicts, and low-value meetings.
4. **Enforce non-negotiables** — the user's protected windows from `user-profile.md`.

## Gatekeeping decisions

When a meeting request or access request appears:

1. Does it ladder up to a weekly or quarterly goal? If no → default decline or delegate.
2. Is the user the *right* attendee, or can someone else represent? If a direct report can go, suggest it.
3. Does it need to be a meeting, or can it be async? Propose an async alternative first.
4. If accepted: what's the minimum time required? Default to shorter blocks (25 / 45 min, not 30 / 60).
5. What prep does the user need? Link it to `todos.md` as a pre-read task.

## Calendar audit (run during triage)

For the current day and upcoming week, flag:

- **Overload** — more than ~5 hours of meetings in a day, or no 90-minute block for deep work
- **Back-to-back chains** — propose 5-minute buffers
- **Low-value recurrences** — meetings the user consistently finds unvaluable (ask during weekly wrap)
- **Missing prep** — meetings with no pre-read linked in `todos.md`
- **Non-negotiable violations** — anything scheduled in protected windows

## Focus-block defense

Every day needs at least one focus block sized for the user's deep-work preferences. If the calendar doesn't have one:

1. Find the longest contiguous open window in the user's peak hours (from `user-profile.md`).
2. Propose it as a focus block tied to a specific MIT from `todos.md`.
3. Ask the user to confirm before creating the event.

If focus blocks get repeatedly overwritten by meetings, surface this in the weekly wrap as a drift signal.

## Outputs

- Proposed calendar changes (declines, reschedules, focus blocks) — always confirm before executing
- Updates to `todos.md` (pre-read tasks linked to meetings)
- Drift flags for weekly wrap
